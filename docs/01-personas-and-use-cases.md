# Personas & Use Cases

## Personas

1. **Product Owner / PM (Primary Buyer)**

   * Goals:

     * Reduce friction, improve adoption.

     * Justify UX/eng improvements with real numbers.

   * Uses:

     * Frustration heatmaps, flow analysis, post-release impact.

2. **Head of UX / Research**

   * Goals:

     * Understand actual behavior patterns vs survey opinions.

     * Prioritize usability work by impact.

   * Uses:

     * Task-flow archetypes, rage-click clusters, segmented behavior.

3. **Data Buyer / Market Analyst (External)**

   * Goals:

     * Understand industry-wide UX patterns.

     * Benchmark their own product against aggregated competition.

   * Uses:

     * "NetCrab Benchmarks" data packs (download / API).

4. **IT / Security Admin**

   * Goals:

     * Maintain compliance.

     * Control what leaves the intranet.

   * Uses:

     * Configure scrubbing, hashing, whitelisting, agent health.

---

## Primary Use Cases

1. **Frustration Radar**

   * "Where do users rage-click, repeat actions, or backtrack?"

   * Output: list of screens/features sorted by **Friction Index**.

2. **Real Task Flow Discovery**

   * "What do users actually do before and after Feature X?"

   * Output: Sankey-style flows + discovered "archetype journeys".

3. **Release Impact Watch**

   * "Did v2.3 make the billing flow worse or better?"

   * Output: before/after friction & efficiency deltas.

4. **NetCrab Marketplace Participation**

   * "Turn my anonymous behavioral exhaust into recurring revenue."

   * Output: revenue reports + visibility into which packs your data powers.

5. **External Benchmark Buying**

   * "What's a good baseline number of clicks and time to complete a support ticket in tools like ours?"

   * Output: dataset / dashboard of cross-product aggregates.
